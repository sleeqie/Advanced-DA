Lab ‘Introduction into PostGIS’

The following is based on the blog Building a Free and Open-Source Spatial Database Using PostgreSQL and PostGIS

Prerequisites

If not installed, install:

  1. PostgreSQL
  2. pgAmin4
  3. QGIS
  4. PostGIS

To install 1., 2. & 4. you can use EDB https://www.enterprisedb.com/downloads/postgres-postgresql-downloads. First, install PostgreSQL and pgAmin4, then use the StackBuilder package manager to install PostGIS (under Spatial Extensions).

Set-up PostgreSQL-DB with pgAdmin

  1. Start pgAmin4 (graphical user interface for interacting with PostgreSQL databases).
  2. Create PostgreSQL DB Server (right click on ‘Servers’) and enter DB informations (tab ‘General’) & credentials (tab ‘Connection’).

  1. Create a schema (right click on ‘Schemas’) in the (default) DB ‘postgres’, name it ‘mod3_demo’.

Load Data into the Database

  1. Download & unzip the ‘GEO_DATA.zip’ folder from the OLAT course.
  2. Navigate to the tab ‘Tools’ & then ‘Query Tool’ to open up a SQL query window in pgAdmin4:

  1. Create an empty table named ‘coffee_shops’ (based on the structure of the ‘coffee_shops.csv’ file) in the schema ‘mod3_demo’ by running following query:
CREATE TABLE mod3_demo.coffee_shops
    (
      id serial NOT NULL,
      name character varying(50),
      address character varying(50),
      city character varying(50),
      state character varying(50),
      zip character varying(10),
      lat numeric,
      lon numeric
    );

  1. Refresh the schema ‘mod3_demo’
  2. Load the ‘coffee_shops.csv’ data in the table by running following query (adjust the path):
COPY mod3_demo.coffee_shops(id,
      name,
      address,
      city,
      state,
      zip,
      lat,
      lon)
FROM '.../GEO_DATA/coffee_shops.csv'
DELIMITER ','
CSV HEADER;
  1. View the data in the table by right click on it -> View Data -> View All Rows.

Set-up PostGIS

  1. To use the ‘PostGIS’ it must be activated (once) by running:
CREATE EXTENSION postgis;
  1. To check (and update) the installed ‘PostGIS’ version run:
ALTER EXTENSION postgis UPDATE;

Make the Data geospatial

  1. Create a ‘geom’ field (column) with the geometry type ‘Point’ and the coordinate reference system ‘WGS84’(EPSG 4326) for the table ‘mod3_demo.coffee_shops’ using following query:
ALTER TABLE mod3_demo.coffee_shops
    ADD COLUMN geom geometry(POINT,4326);
  1. Calculate the geometries (i.e. point locations) based on the columns ‘lat’ & ‘lon’ using the PostGIS function ST_MakePoint(,) and set the coordinate reference system to ‘WGS84’ using ST_SetSRID():
UPDATE mod3_demo.coffee_shops
    SET geom = ST_SetSRID(ST_MakePoint(lon,lat),4326);
  1. Create a spatial index ‘coffee_shops_gist’ on the ‘geom’ field of the table by:
CREATE INDEX coffee_shops_gist
  ON mod3_demo.coffee_shops
  USING gist (geom);
  1. View the geometries of the coffee-shops in the table by right click on it -> View Data -> View All Rows. Go to the column ‘geom’ and click the symbolized eye.

PostGIS use-case examples

  1. Get the geometry type of both tables ‘mod3_demo.coffee_shops’ and ‘mod3_demo.neighborhoods’ using the PostGIS function ST_GeometryType():
SELECT ST_GeometryType(pts.geom) pts_geom, ST_GeometryType(shp.geom) shp_geom
    FROM mod3_demo.coffee_shops pts, mod3_demo.neighborhoods shp
    LIMIT 1;
  1. Number of coffee-shops per neighborhood, i.e. selection by intersection:
SELECT  n.name as name, count(*)
    FROM  mod3_demo.coffee_shops AS cs, mod3_demo.neighborhoods AS n
    WHERE ST_Intersects(cs.geom, n.geom)
    GROUP BY n.name;
  1. Add the neighborhood where the shop is located to the table ‘mod3_demo.coffee_shops’:
ALTER TABLE mod3_demo.coffee_shops
ADD neighborhood;

UPDATE mod3_demo.coffee_shops
    SET neighborhood = n_name
    FROM
        (SELECT cs.id as cs_id, n.name as n_name
        FROM  mod3_demo.coffee_shops AS cs,  mod3_demo.neighborhoods AS n
        WHERE ST_Intersects(cs.geom, n.geom)) x
    WHERE id = cs_id;
  1. Order the coffee-shops by (euclidean) distance to a point (location MIT) with the WGS84 coordinates -71.093500 (lat) & 42.359055 (lon):
SELECT name, address
    FROM coffee_shops
    ORDER BY geom <-> ST_SetSRID(ST_MakePoint(-71.093500,42.359055),4326);
  1. Copy the ‘coffee_shops_utm’ to a new table named ‘coffee_shops_utm’. Reproject the points in ‘mod3_demo.coffee_shops_utm’ and the MIT location to the metric CRS ‘UTM Zone 19N’ (EPSG code 32619). Find all coffee-shops (‘name’ & ‘address’) within 500 m around the MIT:
CREATE TABLE mod3_demo.coffee_shops_utm AS SELECT * FROM mod3_demo.coffee_shops;

ALTER TABLE mod3_demo.coffee_shops_utm
    ALTER COLUMN geom TYPE geometry(POINT, 32619)
    USING ST_Transform(geom, 32619);

SELECT name, address
     FROM mod3_demo.coffee_shops_utm
     WHERE ST_DWithin(geom, ST_Transform(ST_SetSRID(ST_MakePoint(-71.093500,42.359055), 4326), 32619), 500);
  1. Find all coffee-shops (‘name’ & ‘address’) within 500 m around the MIT, add the specific distance, and order the points accordingly:
SELECT name, address, ST_Distance(geom, mit_coord) AS distance
    FROM mod3_demo.coffee_shops_utm, ST_Transform(ST_SetSRID(ST_MakePoint(-71.093500,42.359055), 4326), 32619) AS mit_coord
    WHERE ST_DWithin(geom, mit_coord, 500) ORDER BY distance;
  1. Clustering methods in PostGIS - Create a new table ‘mod3_demo.coffee_shops_utm_cls’ and use the PostGIS functions ‘ST_ClusterKMeans’ to sort the coffee-shops in 3 distance-based clusters using the typical clustering method ‘K-means’:
CREATE TABLE mod3_demo.coffee_shops_utm_cls AS SELECT ST_ClusterKMeans(geom,3) OVER() AS cid, id, geom
    FROM mod3_demo.coffee_shops_utm;
  1. Display the ‘mod3_demo.coffee_shops_utm_cls’ in QGIS.
